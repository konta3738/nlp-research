Год публикации: 2025

В этом документе рассматривается статья о TRepLiNa. В работе предлагается метод, который использует знания из области mechanistic interpretability (механистической интерпретируемости) — анализа внутренних представлений и активаций нейронов языковых моделей — для улучшения качества машинного перевода низкоресурсных языков.

Исследования в области mechanistic interpretability показывают, что многие современные крупные языковые модели, принимая на вход разные языки, преобразуют их внутренние представления в пространство, похожее на английское, и уже затем выполняют дальнейшие рассуждения. Проще говоря: «модель внутри думает на английском языке».

Однако для низкоресурсных языков, которые модель почти не видела ранее, путь от входного языка к «английскому внутреннему представлению» выстроен недостаточно хорошо. TRepLiNa решает эту проблему, добавляя в функцию потерь меры сходства представлений между новым (низкоресурсным) и высокоресурсным языком — в частности, английским. Это помогает модели сформировать надёжный внутренний маршрут.

Эксперименты показывают, что для задач машинного перевода низкоресурсных языков метод часто превосходит обычный fine-tuning, давая лучшие значения BLEU и chrF++ (хотя степень эффекта зависит от близости языков).

В статье основное внимание уделяется направлению низкоресурсный → высокоресурсный язык (например, LRL → английский), однако результаты также указывают на пользу для направления высокоресурсный → низкоресурсный язык.

Принцип работы

В TRepLiNa для контроля степени сходства представлений низкоресурсного и высокоресурсного языков используется функция потерь, объединяющая два компонента:

  • REPINA: регуляризация, подавляющая дрейф представлений между базовой моделью и моделью после fine-tuning
  • CKA (Centered Kernel Alignment): мера сходства структур представлений разных языков

Как работает REPINA

REPINA — это регуляризатор, контролирующий насколько далеко представления после fine-tuning отходят от представлений базовой модели.

Общее представление (REPINA с линейным отображением)

Пусть для некоторого слоя \(\ell\) и токена \(i\):

  • \(h_{\ell,i}^{(0)}\): скрытое состояние базовой модели
  • \(h_{\ell,i}^{(\theta)}\): скрытое состояние модели после fine-tuning

REPINA предполагает существование линейного отображения \(W\) между двумя пространствами. Цель — приблизить \(W h_{\ell,i}^{(\theta)}\) к \(h_{\ell,i}^{(0)}\).

\[ \mathcal{L}_{REPINA} = \frac{1}{N} \sum_{i=1}^{N} \left\lVert W h_{\ell,i}^{(\theta)} - h_{\ell,i}^{(0)} \right\rVert_2^2 \]

REPINAI (identity) в реализации TRepLiNa

В TRepLiNa используется упрощённый вариант — REPINAI, где \(W = I\). Тогда функция потерь принимает вид:

\[ \mathcal{L}_{REPINA^{I}} = \frac{1}{N} \sum_{i=1}^{N} \left\lVert h_{\ell,i}^{(\theta)} - h_{\ell,i}^{(0)} \right\rVert_2^2 \]

То есть **представления модели с LoRA-адаптерами** штрафуются за чрезмерное отклонение от **представлений базовой Aya-23 (с отключёнными адаптерами)**.

Как работает CKA

CKA (Centered Kernel Alignment) — это показатель того, насколько похожа структура двух матриц представлений. Линейная версия CKA (Kornblith et al., 2019) проста в вычислении и удобна для обучения.

Определение линейной CKA

Пусть:

  • \(X \in \mathbb{R}^{N \times D}\) — представления низкоресурсного языка
  • \(Y \in \mathbb{R}^{N \times D}\) — представления высокоресурсного языка

После центрирования по строкам CKA определяется как:

\[ \mathrm{CKA}(X, Y) = \frac{ \left\lVert \tilde{X}^{\top} \tilde{Y} \right\rVert_F^2 }{ \left\lVert \tilde{X}^{\top} \tilde{X} \right\rVert_F \cdot \left\lVert \tilde{Y}^{\top} \tilde{Y} \right\rVert_F } \]

Значение нормировано: \(0 \le \mathrm{CKA} \le 1\). Чем ближе к 1, тем ближе структуры представлений.

TRepLiNa минимизирует следующую величину:

\[ \mathcal{L}_{CKA} = 1 - \mathrm{CKA}(X, Y) \]

Важно отметить, что CKA может **исказить** внутренние представления высокоресурсного языка, если их чрезмерно подгонять под низкоресурсный язык. Поэтому в статье подчёркивается необходимость совместного использования CKA и REPINA.

1. Реализация CKA

def center_rows(x: torch.Tensor, mask: Optional[torch.Tensor] = None) -> torch.Tensor:
    """
    Center features along the sample dimension (rows).

    x: [N, D]
    mask: [N] boolean or float mask (1 for keep). If provided, compute mean over masked rows.
    """
    if mask is None:
        mean = x.mean(dim=0, keepdim=True)
        return x - mean
    else:
        if mask.dtype != torch.float32:
            mask = mask.float()
        # avoid division by zero
        denom = mask.sum().clamp(min=1.0)
        mean = (x * mask.unsqueeze(1)).sum(dim=0, keepdim=True) / denom
        return x - mean
def linear_cka(x: torch.Tensor, y: torch.Tensor,
               x_mask: Optional[torch.Tensor] = None,
               y_mask: Optional[torch.Tensor] = None,
               eps: float = 1e-6) -> torch.Tensor:
    """
    Compute linear CKA between two representation matrices.

    x: [N, D]
    y: [M, D]  (we'll downsample/upsample to min(N,M) by truncation to align tokens if not equal)
    masks: optional [N] and [M] masks
    """
    # If different number of rows (tokens), align by truncation to min length.
    n = x.shape[0]
    m = y.shape[0]
    L = min(n, m)
    x = x[:L]
    y = y[:L]
    if x_mask is not None:
        x_mask = x_mask[:L]
    if y_mask is not None:
        y_mask = y_mask[:L]

    # Center rows
    x = center_rows(x, x_mask)
    y = center_rows(y, y_mask)

    # Following Kornblith et al. (2019): CKA = ||X^T Y||_F^2 / (||X^T X||_F * ||Y^T Y||_F)
    # where X, Y are row-centered.
    xty = x.T @ y
    num = (xty * xty).sum()

    xtx = x.T @ x
    yty = y.T @ y
    denom = torch.sqrt((xtx * xtx).sum().clamp(min=eps)) * \
            torch.sqrt((yty * yty).sum().clamp(min=eps))
    return (num / denom).clamp(min=0.0, max=1.0)

3. Общая функция потерь

def compute_losses(
    model,
    batch: Dict[str, torch.Tensor],
    tokenizer,
    layer_index: int,
    lambda_cka: float,
    mu_repina: float,
    bf16: bool,
    global_step
):
    """
    Returns (total_loss, dict of components)
    """
    device = next(model.parameters()).device
    repina_loss = torch.tensor(0.0, device=device)

    # 1) Task loss: LM on prompt+target (labels already mask prompt with -100)
    input_ids = batch["input_ids"].to(device)
    labels    = batch["labels"].to(device)
    attention_mask = (input_ids != tokenizer.pad_token_id).to(device)

    outputs = model(
        input_ids=input_ids,
        attention_mask=attention_mask,
        labels=labels,
        use_cache=False,
        output_hidden_states=True
    )
    task_loss = compute_task_loss_with_label_smoothing(
        outputs, labels, epsilon=0.1, ignore_index=-100
    )

    # 2) Alignment loss (CKA) on source regions of A and B
    a_ids = batch["a_align_ids"].to(device)
    b_ids = batch["b_align_ids"].to(device)

    a_attn = (a_ids != tokenizer.pad_token_id).to(device)
    b_attn = (b_ids != tokenizer.pad_token_id).to(device)

    a_out = forward_hidden(model, a_ids, a_attn)
    b_out = forward_hidden(model, b_ids, b_attn)

    a_reps_all, a_tokmask_all = gather_layer_hidden(a_out.hidden_states, layer_index, a_attn)
    b_reps_all, b_tokmask_all = gather_layer_hidden(b_out.hidden_states, layer_index, b_attn)

    a_src_keep = a_attn.reshape(-1).bool()
    b_src_keep = b_attn.reshape(-1).bool()

    a_src = a_reps_all[a_src_keep]
    b_src = b_reps_all[b_src_keep]

    if a_src.numel() == 0:
        a_src = a_reps_all[a_tokmask_all]
    if b_src.numel() == 0:
        b_src = b_reps_all[b_tokmask_all]

    cka_val = linear_cka(a_src, b_src, None, None)
    cka_loss = 1.0 - cka_val

    # 3) REPINA^I
    do_repina = ((global_step + 1) % 2 == 0)
    if do_repina:
        with torch.no_grad():
            with model.disable_adapter():
                base_out = forward_hidden(model, a_ids, a_attn)
        base_reps_all, _ = gather_layer_hidden(base_out.hidden_states, layer_index, a_attn)
        rep_cur = a_reps_all[a_src_keep]
        rep_pre = base_reps_all[a_src_keep]
        if rep_cur.shape[0] != rep_pre.shape[0]:
            L = min(rep_cur.shape[0], rep_pre.shape[0])
            rep_cur = rep_cur[:L]
            rep_pre = rep_pre[:L]

        repina_loss = torch.mean((rep_cur - rep_pre) ** 2)

        total = task_loss + lambda_cka * cka_loss + mu_repina * repina_loss
    else:
        total = task_loss + lambda_cka * cka_loss

    return total, {
        "task_loss": task_loss.detach().float().item(),
        "cka": cka_val.detach().float().item(),
        "cka_loss": cka_loss.detach().float().item(),
        "repina": repina_loss.detach().float().item(),
        "total": total.detach().float().item()
    }

Итоговая формула

Обозначим: \[ \mathcal{L}_{task},\quad \mathcal{L}_{CKA} = 1 - CKA(X,Y),\quad \mathcal{L}_{REPINA^I} \] Тогда общая функция потерь: \[ \mathcal{L}_{total} = \mathcal{L}_{task} + \lambda_{CKA} \cdot \mathcal{L}_{CKA} + \mu_{REPINA} \cdot \mathcal{L}_{REPINA^I} \] (На практике REPINA вычисляется только каждые несколько шагов.)
